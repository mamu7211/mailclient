# Feature 7.1: Internationalization (i18n)

## Goal

Support multiple languages across the entire UI with en_US as the neutral/default language.

## Languages

| Locale | Language | Role |
|--------|----------|------|
| `en_US` | English (US) | Default / neutral language |
| `de_DE` | German | Foreign language |
| `fr_FR` | French | Foreign language |
| `it_IT` | Italian | Foreign language |

## Deliverables

### Infrastructure

- .NET resource files (`.resx`) for each supported locale
- `IStringLocalizer<T>` integration in Blazor components
- Language detection: browser `Accept-Language` header → user preference → fallback `en_US`
- Language switcher component in UI

### API

- `Accept-Language` header support for API error messages
- Localized validation error messages

### UI

- All static text extracted to resource files
- Language switcher (dropdown or icon) in the layout
- User language preference persisted (localStorage or user settings)
- RTL support not required (none of the target languages need it)

### Shared

- Localized display names on DTO validation attributes

## Acceptance Criteria

- [ ] All UI text rendered from resource files, no hardcoded strings
- [ ] Switching language updates all visible text without page reload
- [ ] en_US is the fallback when a translation key is missing
- [ ] API error messages respect `Accept-Language` header
- [ ] All four locales (en_US, de_DE, fr_FR, it_IT) have complete translations
